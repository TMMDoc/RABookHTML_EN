Home Parameter

Before proceeding to the home parameter, the homing procedure is explained below.

Homing Introduction

Most encoders used these days are incremental encoders which can only report changes in position not the absolute position. Hence for an encoder interface to track and report absolute position, the encoder counts must be correlated to a reference position in the mechanical system to which the encoder is attached. This is commonly done by homing the system, which consists of moving the mechanical system (and encoder) until it aligns with a reference position.

Homing is required whenever machine/controller restarts or when some machine parameters are changed.

Homing Procedure in RA

Homing can be done in RA software either in JOG screen or RUN screen.

When an axis is not homed the position shown is prefixed with a '*', indicating that position is not correct, and axis needs to be homed.

When user enters RUN screen and axes are not homed, then 'Machine not referenced' message appears in the bottom right of the screen.

  1. Make sure the emergency button is released and pump is ON.

  2. After reaching the RUN screen 'Machine not referenced' message appears.

    Case 1 (RAM position below reference mark):

  3. Press the Start button, RAM moves up in search of reference mark and backgauge axis also starts homing in sequence entered in machine parameters (Axis parameters → Homing parameter → Sequence).

  4. The '*' prefix disappears for axes position (if homed) in numeric screen.

  5. Now if reset (UP pedal) is pressed the backgauge moves to commanded position and program is ready to RUN.

    Case 2 (RAM position above reference mark):

  6. Press the Start button, RAM moves up in search of reference mark and backgauge also starts to home in sequence entered in machine parameters. As the reference mark is below the RAM, RAM moves up till mechanical upper dead point (beyond this point lag error increases as despite giving command RAM will not be able to move) and stops as no reference pulse is received.

  7. Now message 'Press Down pedal until RAM stops' appears.

  8. RAM moves down in search of reference mark as down pedal is pressed.

  9. While RAM moves down and the reference mark is found, the homing process of RAM (Y1, Y2) is not completed as it is advised to obtain the reference signal when RAM moves in same direction (UP in this case). Now the controller goes to stop mode and STOP LED (RED LED) glows in the panel.

  10. Now message 'Press Start button' appears. When Start button is pressed RAM moves up and completes homing after finding reference mark while moving in UP direction.

  11. Now pressing UP pedal (reset) the backgauge moves to the target position and program is ready to RUN.

Homing can also be done from JOG mode following the same procedure.

Homing Direction change

In case the operator knows that RAM is above the reference mark and wants to avoid the default upward movement of the RAM then operator can force a change in homing direction by following these steps:

  1. Enter RUN or JOG screen, emergency button must be in released state and pump must be ON.

  2. Now make sure machine is in stop condition (RED LED must glow in panel), if not in stop condition press the stop button and then press down pedal for 5 seconds.

  3. 'Homing dir down, release foot pedal' message appears. Release the foot pedal.

  4. Now 'Press down pedal until RAM stop' message appears.

  5. Press the Start button and then press down pedal.

  6. RAM stops after detecting the reference pulse, homing is not yet completed as the reference pulse must always be found when is moving UP.

  7. 'Press start button' message appears. Now after pressing start button RAM moves UP, finds the reference pulse and homing will complete.

For machine operators

While homing (when RAM moves down in search for reference pulse) if the footpedal down is not pressed continuously or the RAM movement is unstable there are chances for not detecting the reference pulse in this case RAM keeps moving down and if tools are mounted in the machine it may collide.

Hence operator must monitor the RAM when moving down and in cases where operator feels the reference pulse has not been detected despite RAM moving down significantly, operator must stop pressing down pedal. Operator must stop pressing Down pedal and press the emergency button, then release the emergency and press reset (Up pedal) to ON the pump, and again press reset (Up pedal). Now on pressing the start button, homing will start with RAM moving up (default homing direction).

RAM synchronization error

In some cases, due to unstable movement the Y1 Y2 difference exceeds the allowed Y1 Y2 difference and RAM Sync error occurs and RAM cannot be moved in RUN or JOG mode. In these cases, it is required to login to machine parameters with service access level and enter diagnostic mode and manually move the RAM, hence operators cannot handle this error on their own. Hence in RA there is a feature provided which forces homing to happen for Y1 Y2 (without restarting the machine), so that operators can handle RAM sync error without resorting to service person.

How to handle RAM Synchronization error

  1. Once RAM Sync error occurs, it is not possible to move RAM in run mode.

  2. Now operator must press emergency button and press Up pedal for about 5 sec until the Y axis position is prefixed with *. This means homing for Y axis will start once again.

  3. Now operator can release the emergency, press reset (Up pedal) to ON the pump and press start button to start the normal homing process.

  4. Now RAM moves up in search reference mark and normal homing process starts as mentioned above.

Conclusion

As homing when RAM position is above the reference mark is a two-step process, it is advised to lower the RAM before restarting or powering off the machine in order to complete homing quickly when machine is powered on next time.

Generally, when the homing command is activated, all the backgauge axis will start to move in the homing direction (Safe direction, selected in homing parameter). If the backgauge is near to the tools during homing, then the movement of the R - axis may result in collision with tools. In order to avoid this type of collision the X-axis will be homed first with Sequence number as 1 (default value of parameter) and other R & Z axis will be sequence number 2 (default value of parameter), to follow the homing process.
